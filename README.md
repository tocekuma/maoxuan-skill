# 毛选 · 思维操作系统 (Mao's Selected Works Thinking OS)

> 用《毛泽东选集》五卷的认知框架分析商业竞争、产品决策、组织管理问题。

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

## 这是什么？

一个面向 AI Coding Assistant（Cursor / Claude Code）的 **Skill**，将毛泽东选集中的核心思维方法论提炼为可操作的分析框架，帮助你用第一性原理分析复杂商业问题。

**适用场景**：竞争策略、市场进入、团队冲突、组织变革、产品决策、危机应对——尤其是**以弱胜强**的非对称竞争。也适用于**个人成长**：职业规划、技能学习、人际关系、习惯养成、低谷突围、时间管理。

## 核心特性

| 特性 | 说明 |
|------|------|
| 🎭 双模式 | **教员模式**（角色扮演，第一人称）+ **方法论模式**（结构化分析报告） |
| 🔬 五步分析引擎 | 调查摸底 → 矛盾定位 → 力量评估 → 策略制定 → 验证回路 |
| 📚 12 个方法论模型 | 矛盾分析、持久战略、农村包围城市、统一战线、群众路线等 |
| 📖 原文论证链 | 每个框架配有《毛选》原文段落作为论证依据 |
| 🗺️ 情境路由器 | 根据问题类型自动匹配最佳框架组合 |
| 💼 商业案例库 | 拼多多、字节跳动、华为等企业的毛选式战略解读 |

## 安装

### Cursor

```bash
# 克隆到 Cursor skills 目录
git clone https://github.com/tocekuma/maoxuan-skill.git ~/.cursor/skills/maoxuan
```

### Claude Code

```bash
# 克隆到 Claude Code skills 目录
git clone https://github.com/tocekuma/maoxuan-skill.git ~/.claude/skills/maoxuan
```

### 手动安装

1. 下载本仓库的 ZIP 文件
2. 解压到 `~/.cursor/skills/maoxuan/`（Cursor）或 `~/.claude/skills/maoxuan/`（Claude Code）
3. 确保目录结构如下：

```
maoxuan/
├── SKILL.md              # 主文件（路由器 + 分析引擎 + 双模式切换）
└── references/
    ├── 00-analysis-engine.md       # 五步分析法详细操作清单
    ├── 01-contradiction-method.md  # 矛盾分析法
    ├── 02-practice-theory.md       # 实践论 + 调查研究
    ├── 03-protracted-strategy.md   # 持久战略 + 军事原则
    ├── 04-periphery-to-center.md   # 农村包围城市 + 根据地
    ├── 05-united-front.md          # 统一战线 + 敌我友分析
    ├── 06-mass-line.md             # 群众路线 + 领导方法
    ├── 07-dialectics-toolkit.md    # 辩证法工具箱
    ├── 08-organizational-theory.md # 组织建设论
    ├── 09-expression-dna.md        # 教员模式表达风格 DNA
    ├── 10-business-cases.md        # 商业应用案例库
    └── 11-personal-growth.md       # 个人成长场景库
```

## 使用方法

安装后，在 Cursor 或 Claude Code 对话中直接使用触发词：

### 教员模式（角色扮演）

```
教员，我们公司是一家小型 SaaS，现在被大厂抄袭了功能，怎么办？
```

```
以教员的视角分析一下，团队里两个技术方案争执不下该如何处理？
```

### 方法论模式（结构化分析）

```
用毛选分析一下我们的市场竞争策略
```

```
毛选方法论：我想进入一个新市场但资源有限，应该怎么规划？
```

## 12 个方法论模型

| # | 模型 | 一句话本质 | 适用场景 |
|---|------|----------|---------|
| 1 | 矛盾分析法 | 找到主要矛盾 = 找到破局点 | 问题太多不知从何下手 |
| 2 | 实践认识循环 | 别在会议室讨论三个月，先做 MVP 试 | 方案争执不下 |
| 3 | 持久战略 | 敌强我弱时以时间换空间 | 面对远大于自己的竞争对手 |
| 4 | 农村包围城市 | 先在边缘建根据地，再包围中心 | 新市场进入、下沉策略 |
| 5 | 统一战线 | 把朋友搞得多多的，敌人搞得少少的 | 联盟、合作、谈判 |
| 6 | 群众路线 | 最好的需求来自一线用户反馈的系统化提炼 | 产品决策、团队管理 |
| 7 | 纸老虎论 | 战略藐视，战术重视 | 面对看似强大的对手 |
| 8 | 辩证法工具箱 | 好事有危机，坏事有转机 | 危机应对、形势判断 |
| 9 | 集中优势兵力 | 资源不够就缩小战场 | 资源有限时的聚焦决策 |
| 10 | 调查研究 | 没有调查就没有发言权 | 做决策前的信息收集 |
| 11 | 整风方法论 | 团结-批评-团结 | 组织变革、团队复盘 |
| 12 | 论十大关系 | 不是选 A 或 B，是找平衡点 | 多重矛盾需要平衡 |

## 技术架构

```
SKILL.md（270行）
  ├── YAML frontmatter — 描述 + 触发词
  ├── 双模式系统 — 教员模式 / 方法论模式
  ├── 五步分析引擎 — 核心分析流程
  ├── 情境路由器 — 问题类型 → 框架匹配
  ├── 12 模型速查 — 一句话 + 商业翻译
  ├── 输出格式 — 两种模式的输出模板
  ├── 决策启发式 — 10 条快速参考
  ├── 反模式警告 — 7 类思维陷阱
  └── Reference 索引 — 按需加载
references/（11个文件，~70KB）
  └── 每个框架的深度操作指南 + 《毛选》原文段落
```

**设计原则**：SKILL.md 保持精简（<300行），作为路由器快速匹配场景；详细内容按需从 references/ 加载（Progressive Disclosure），避免一次性塞入太多上下文。

## 与 leezythu/maoxuan-skill 的区别

| 维度 | leezythu 版 | 本项目 |
|------|------------|--------|
| 深度 | 概念卡片式 | 每个框架有完整操作步骤 + 判断标准 |
| 原文 | 少量引用 | 大量《毛选》原文段落构成论证链 |
| 可操作性 | 启发性为主 | 五步分析引擎 + 情境路由器 + 输出模板 |
| 商业应用 | 通用 | 侧重商业/创业场景，配有企业案例库 |
| 双模式 | 角色扮演 | 角色扮演 + 结构化分析双模式切换 |
| 架构 | 单文件 | SKILL.md 路由 + 11 个 reference 文件（渐进式加载） |
