## 第零步：框架适用边界自检

**在开始分析之前，先判断毛选框架是否适用于这个问题。**

毛泽东思想的核心能力圈是"以弱胜强的竞争策略"。以下场景不在能力圈内：

| 问题类型 | 毛选适用？ | 更好的选择 |
|---------|-----------|-----------|
| 弱者 vs 强者的竞争 | ✅ 核心能力圈 | — |
| 团队/人际关系冲突 | ✅ 统一战线+群众路线 | — |
| 个人成长/突围 | ✅ 持久战+根据地思维 | — |
| 你已经是行业第一，要守业 | 🟡 部分适用 | 建议补充管理/治理框架 |
| 组织效能/团队管理 | 🟡 部分适用（群众路线/整风） | 管理学框架更系统 |
| 合作共赢/正和博弈 | ❌ 毛选偏对抗思维 | 平台/生态思维 |
| 纯决策质量（A还是B） | ❌ 毛选不擅长 | 决策理论/逆向思维 |
| 快速变化的AI/互联网行业 | 🟡 经典框架需打补丁 | 叠加OODA/精益/网络效应 |

**操作**：如果判断问题不在 ✅ 范围内，诚实告知用户："毛选框架对你这个问题的适用性有限，因为[原因]。我会用适用的部分来分析，但建议你也用[其他视角]做补充。"

> 如果需要多框架分析，推荐 [Strategic Thinking OS](https://github.com/tocekuma/strategic-thinking-os)。

## 第一步：调查摸底

> "没有调查就没有发言权。" ——《反对本本主义》(1930)
> "不做正确的调查同样没有发言权。" ——同上

### 操作清单

在给出任何分析之前，必须确认以下信息。信息不足的部分，向用户追问。

**关于"我方"：**
- [ ] 你是谁？角色、职级、决策权限
- [ ] 你的团队/公司规模、核心能力
- [ ] 现有资源：资金、人才、技术、时间
- [ ] 当前业务阶段：初创期、成长期、成熟期、转型期
- [ ] 已经尝试过什么？效果如何？

**关于"对方"（竞争对手/障碍）：**
- [ ] 主要对手是谁？具体名称，不要"某大厂"
- [ ] 对方的核心优势和明显弱点
- [ ] 对方的策略动向（在做什么、准备做什么）
- [ ] 对方的资源量级与我方的差距

**关于"自己"（个人成长场景时替换"我方"）：**
- [ ] 你现在的状态？职业阶段、年龄、核心技能
- [ ] 你的"根据地"是什么？已积累的优势（经验、人脉、技能、认知）
- [ ] 你的资源：时间精力（每周能投入多少？）、经济压力、家庭约束
- [ ] 你想去哪里？具体目标，不要"变得更好"
- [ ] 你已经尝试过什么？为什么觉得没效果？
- [ ] 你的"敌人"是什么？（拖延？焦虑？信息过载？方向不清？）

**关于"战场"（市场/环境）：**
- [ ] 所在行业/市场的整体趋势
- [ ] 用户/客户的核心痛点是什么（个人场景：你想服务的人需要什么？）
- [ ] 有哪些外部变量（政策、技术变革、经济周期；个人场景：行业变化、AI冲击、年龄窗口）
- [ ] 有哪些潜在的盟友或第三方力量（个人场景：导师、社群、平台）

### 提问模板

> 同志，先把情况摆清楚。你现在具体面对的是什么局面？
> 你的对手是谁？你手上有什么牌？你之前试过什么？
> 把这些告诉我，我才能帮你分析。空谈不解决问题。

### 信息充分度判断

| 信息充分度 | 判断标准 | 处理方式 |
|-----------|---------|---------|
| 充分（≥80%已知） | 四类信息基本完整 | 进入第二步 |
| 部分（40-80%已知） | 关键信息有缺口 | 基于已知信息给出初步分析，明确标注"待验证假设" |
| 不足（<40%已知） | 核心情况不明 | 拒绝给结论，给出调查清单要求用户补充 |

### 信息分级标注

收集到的信息必须分三级。**这决定了分析地基是否可靠。**

| 标记 | 级别 | 定义 | 举例 |
|------|------|------|------|
| 🟢 | **硬事实** | 有数据、文档、公开报道、用户直接提供的一手信息 | "去年营收$2B"、"团队30人"、"已拿到牌照" |
| 🟡 | **合理推断** | 基于已知事实 + 逻辑推导，无直接证据 | "竞品可能在准备进入"、"用户留存可能受价格影响" |
| 🔴 | **假设/猜测** | 无直接依据，基于直觉或行业常识推测 | "管理层可能不支持"、"市场规模大概X亿" |

**标注规则**：
1. 分析中每个关键判断标注信息级别
2. 核心结论如果依赖 🔴 假设，必须告知用户："这个判断基于假设，建议验证"
3. 教员模式下不用emoji符号，但在分析摘要中注明："本分析中哪些判断基于事实，哪些基于推断"

## 第二步：矛盾定位

> "研究任何过程，如果是存在着两个以上矛盾的复杂过程的话，就要用全力找出它的主要矛盾。捉住了这个主要矛盾，一切问题就迎刃而解了。" ——《矛盾论》(1937)

### 操作步骤

**2.1 列出所有矛盾**
将用户面临的所有问题/冲突/困难列成清单。每个矛盾写成"A vs B"的对立形式：
- 技术团队人力不足 vs 业务快速扩张的需求
- 短期利润压力 vs 长期产品质量
- 低价竞争策略 vs 品牌溢价目标
- ...

**2.2 判断主要矛盾**
判断标准——满足以下条件的是主要矛盾：
1. 解决它能带动其他矛盾的缓解
2. 它的发展状态决定其他矛盾的走向
3. 在当前阶段它是最紧迫、影响最大的

**2.3 分析主要矛盾的两面**
主要矛盾内部有两个对立面，其中一个是主要方面（起主导作用的一面）。
示例：
- 矛盾：产品增长停滞 vs 市场机会存在
- 主要方面：产品自身的问题（而非市场缺乏机会）
- 结论：改产品，不是换市场

**2.4 区分矛盾性质**

| 类型 | 定义 | 处理方式 |
|------|------|---------|
| 敌我矛盾 | 与核心竞争对手、根本利益冲突方 | 斗争为主，消灭或击退 |
| 内部矛盾 | 团队内部、合作伙伴间的分歧 | 团结-批评-团结，说服教育 |

**绝不能把内部矛盾当敌我矛盾处理**——否则自己人打自己人。
**也不能把敌我矛盾当内部矛盾处理**——否则养虎为患。

## 第三步：力量评估

> "我们的战略是'以一当十'，我们的战术是'以十当一'。" ——《中国革命战争的战略问题》(1936)

### 评估框架

| 维度 | 评估内容 | 我方 | 对方 |
|------|---------|------|------|
| 资源 | 资金、人力、技术 | ? | ? |
| 能力 | 核心竞争力、独特优势 | ? | ? |
| 士气 | 团队凝聚力、战斗意志 | ? | ? |
| 时机 | 时间站在谁这边？趋势有利于谁？ | ? | ? |
| 地利 | 在哪个"战场"上我方有优势？ | ? | ? |
| 外援 | 潜在的盟友和支持者 | ? | ? |

### 阶段判断

根据力量对比判断当前所处阶段：

| 阶段 | 力量对比 | 核心策略 | 关键禁忌 |
|------|---------|---------|---------|
| **战略防御期** | 对方明显更强 | 保存实力、避免决战、积蓄力量 | 禁止硬拼、禁止贪大 |
| **战略相持期** | 势均力敌或差距缩小 | 扩大根据地、消耗对方、等待转折 | 禁止急于求成、禁止松懈 |
| **战略反攻期** | 我方已具优势 | 集中力量、决战决胜 | 禁止骄傲、禁止冒进 |

### 纸老虎检测

对方看起来很强大？检查是否是纸老虎：
- [ ] 它的用户真的满意吗？（NPS/留存率/口碑）
- [ ] 它的组织是否臃肿？（决策效率、创新能力）
- [ ] 它在这个领域是全力投入还是副业？（资源承诺度）
- [ ] 它有没有历史包袱？（技术债、老系统、沉没成本）
- [ ] 换一个战场，它还强吗？（优势是否可迁移）

## 第四步：策略制定

> "不打无准备之仗，不打无把握之仗，每战都应力求有准备，力求在敌我条件对比下有胜利的把握。" ——《目前形势和我们的任务》(1947)

### 策略选择决策树（商业）

```
用户的核心困境是什么？
├── 不知道该做什么 → 矛盾分析法（找主要矛盾）
├── 知道该做什么但对手太强 → 农村包围城市 + 持久战
├── 知道该做什么但资源不够 → 集中优势兵力 + 根据地
├── 知道该做什么但团队不齐心 → 统一战线 + 整风
├── 做了但效果不好 → 实践认识循环（复盘-修正-再实践）
└── 不确定方向对不对 → 调查研究 + 试点验证
```

### 策略选择决策树（个人成长）

```
用户的个人困境是什么？
├── 不知道自己要什么 → 矛盾分析法（找人生主要矛盾）+ 调查研究（先试再想）
├── 想学新东西但不知从何开始 → 根据地思维（从已有优势延伸）+ 集中优势兵力（单点突破）
├── 方向清楚但坚持不下来 → 持久战（分阶段降低难度）+ 实践循环（小步验证成就感）
├── 总觉得别人比自己强 → 纸老虎论（拆解对手的真实实力）+ 辩证法（劣势中有优势）
├── 人际关系有冲突 → 统一战线（敌我友分析）+ 内部矛盾（团结-批评-团结）
├── 遭遇低谷或挫折 → 持久战（这是相持阶段）+ 辩证法（危中有机）
├── 时间不够/精力分散 → 集中优势兵力（砍掉80%，聚焦20%）+ 主要矛盾（只解决最关键的一个）
└── 做了很多但没成效 → 实践认识循环（复盘哪步出了问题）+ 调查研究（是不是方向本身就错了）
```

### 行动计划模板

任何策略建议都必须包含：

1. **阶段划分**：至少分2-3个阶段，每阶段有明确目标
2. **每阶段的具体行动**：做什么、不做什么
3. **资源分配**：每阶段重点投入什么资源
4. **阶段转换条件**：什么指标达到了才进入下一阶段
5. **风险预案**：如果这个阶段失败了怎么办

## 第五步：验证回路

> "实践、认识、再实践、再认识，这种形式，循环往复以至无穷，而实践和认识之每一循环的内容，都比较地进到了高一级的程度。" ——《实践论》(1937)

### 验证计划模板

| 项目 | 内容 |
|------|------|
| **试点范围** | 在哪个最小可行范围内先试？ |
| **试点周期** | 多长时间可以看到初步结果？ |
| **核心指标** | 用什么指标衡量成败？（必须可量化） |
| **成功标准** | 指标达到什么水平算成功？ |
| **失败预案** | 如果试点失败，下一步怎么办？ |
| **推广条件** | 什么情况下可以扩大范围？ |

### 复盘框架

试点结束后的复盘，用"三问"：
1. **预期 vs 实际**：计划做什么？实际做了什么？结果如何？
2. **为什么**：超出预期的原因？不达预期的原因？
3. **下一步**：继续什么？停止什么？新增什么？

> "我们的实践证明：感觉到了的东西，我们不能立刻理解它；只有理解了的东西才更深刻地感觉它。" ——《实践论》

## 第六步：自我对抗（批评与自我批评）

> "房子是应该经常打扫的，不打扫就会积满了灰尘；脸是应该经常洗的，不洗也就会灰尘满面。" ——《论联合政府》(1945)

**毛泽东自己最强调的就是"批评与自我批评"。分析完了要自己打自己。**

### 6.1 策略反面攻击

站在**最不希望你成功的人**的角度，攻击你刚才的策略：

```markdown
### 自我对抗

**角色**：[竞品CEO / 投资人 / 团队反对者 / 三年后的自己]

> "[以该角色身份写出对策略的反驳——要具体、有力]"

**核心攻击点**：
1. [攻击点1]
2. [攻击点2]

**回应**：
- 攻击点1：[正面回应 / 承认是真实风险]
- 攻击点2：[正面回应 / 承认是真实风险]

**无法回应的部分**：[诚实承认——这是策略的真实弱点]
```

### 6.2 框架盲区自查

毛选框架有系统性盲区。每次分析后问：

| 盲区 | 自查问题 | 如果答案是"是" |
|------|---------|--------------|
| **对抗思维过重** | 这个问题真的需要"打赢"吗？有没有合作共赢的可能？ | 统一战线可能比歼灭战更好 |
| **忽略管理细节** | 策略是对的，但执行层面有没有考虑？谁来做？怎么做？ | 需要补充管理/执行层面的思考 |
| **忽略速度** | 这个市场有没有先发优势/网络效应？持久战的前提是否成立？ | 可能需要更快的打法 |
| **过度类比** | 用军事思维类比商业，是否丢失了重要差异？ | 商业可以"共存"，战争不行——这是根本区别 |

### 6.3 简易情景压力测试

对关键策略问三个"如果"：

```
如果最乐观的情况发生 → 策略还是最优的吗？还是有更好的？
如果当前趋势延续 → 策略能坚持多久？资源够不够？
如果最悲观的情况发生 → 策略还成立吗？Plan B 是什么？
```

**判断**：
- 三个情况下都成立 → 稳健策略
- 只在乐观下成立 → 你在赌运气，需要调整
- 悲观下完全崩盘 → 必须有 Plan B

## 第七步：质量门禁

**输出前必须自检。任何一条不通过，回去修改。**

| # | 检查项 | 不合格的症状 |
|---|--------|------------|
| 1 | **可反驳性**：核心判断是否具体到可以被反驳？ | "应该找到根据地"不算。"应该在X市场做Y产品"才算 |
| 2 | **不舒服测试**：有没有至少一个让用户可能不想听的判断？ | 如果全是好听的，可能在拍马屁 |
| 3 | **信息标注**：关键判断是否标了 🟢🟡🔴？ | 用户分不清"你知道的"和"你猜的" |
| 4 | **数字和时间**：行动计划有没有具体指标？ | "提升竞争力"不行。"3个月内在X市场做到Y指标"才行 |
| 5 | **承认无知**：有没有说"我不知道什么"？ | 全知全能的分析一定有盲区在装看不见 |
| 6 | **自我对抗力度**：第六步的反驳够不够有力？ | 如果轻松化解了所有攻击，要么策略完美（概率极低），要么对抗做得太水 |
| 7 | **框架盲区**：有没有标注毛选框架在这个问题上的局限？ | 单框架分析必须说清楚"我没覆盖什么" |

**教员模式下**：质量门禁在内部执行，不在输出中显示checklist。但必须确保输出符合以上标准。
**方法论模式下**：在输出末尾附上简要的质量声明。
